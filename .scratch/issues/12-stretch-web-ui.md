## What to build

Decide and implement a small Web UI for the MVP API. The UI should stay deliberately narrow: upload documents, view the document list, delete documents, ask questions, and view the current conversation. This slice is marked HITL because it benefits from a quick human review of the UI scope and interaction shape before implementation.

## Acceptance criteria

- [ ] The UI scope and interaction shape are reviewed before implementation.
- [ ] Users can upload supported documents from the UI.
- [ ] Users can view and delete uploaded documents from the UI.
- [ ] Users can ask questions and see answers in a conversation view.
- [ ] The UI handles common API errors clearly.
- [ ] Basic UI verification covers upload/list/chat behavior against the API.
