## What to build

Create the initial FastAPI service skeleton for the Document Q&A System. The slice should make the project runnable, testable, and ready for later document, retrieval, and chat features. Include configuration loading, a basic application entrypoint, a health check endpoint, and the initial test scaffold.

## Acceptance criteria

- [ ] The project has a runnable FastAPI application with `GET /health`.
- [ ] Configuration can be loaded from environment variables with sensible local defaults.
- [ ] The test suite can run locally and includes coverage for the health endpoint.
- [ ] The repository has a clear Python project structure for API routes, services, persistence, and tests.

## Blocked by

None - can start immediately
